# Demo video script — English (2:48 target)

The narration is 258 words. Every exercise shown comes from the submitted
library of 132 original pre-authored exercises.

## 0:00–0:16 — Problem and insight

**Visual:** AIterval title card and Education category.

**Narration:** At an international summer school, I struggled to follow lectures
and conversations in English. As a researcher, I had little dedicated study
time. Meanwhile, AI tools repeatedly made me wait.

## 0:16–0:58 — Core product flow

**Visual:** Signed-out judge demo. Start the simulated wait, play one exercise,
select an answer, show feedback, and complete the sprint so recovered time rises.

**Narration:** AIterval connects them. It observes only semantic generation-state
controls—never my prompt or the response. After the threshold, one pre-authored
listening exercise appears. I play the sentence, choose an answer, get clear
feedback, and turn lost seconds into recovered learning time.

## 0:58–1:30 — v0.2.1 AI-ready behavior

**Visual:** Released v0.2.1 extension loaded from its verified ZIP on the
deterministic supported-site fixture. Start speech, mark the host ready, hold the
readable notice while playback remains active, then finish and answer. Keep
**Return to AI now** visible.

**Narration:** Here is the released Manifest V3 extension in a deterministic
fixture. The host AI becomes ready while speech is still playing. The readable
notice appears, but audio continues naturally and every exercise control remains
available. I can finish the question or use Return to AI now.

## 1:30–1:55 — Dashboard and learning design

**Visual:** Released extension dashboard with deterministic local sessions. Show
recovered time, completed exercises, review queue, weak skills, preferences, and
voice selection. Keep the archived lecture surface hidden.

**Narration:** Those tiny sessions form a learning system: recovered time,
completed exercises, a review queue, weak skills, difficulty and topic
preferences, and English voice selection. All 132 original, pre-authored
exercises work locally without an API key.

## 1:55–2:15 — Privacy

**Visual:** Public privacy page, including collection boundaries and runtime
scope.

**Narration:** Privacy is a product constraint. AIterval never reads prompts or
responses. It observes only narrow generation-state controls. Learning progress
remains local, with no analytics, advertising, tracking SDK, or remote extension
code.

## 2:15–2:38 — Codex and GPT-5.6 development

**Visual:** Public repository, architecture documentation, tests, and green CI.

**Narration:** I supplied the lived problem, core interaction, product direction,
privacy boundaries, and final design decisions. Codex with GPT-5.6 implemented
and refined the state machine, browser adapters, interface, accessibility,
tests, deployment, and release workflow. The public repository, architecture
notes, tests, and green continuous integration make that inspectable. GPT-5.6
supported development; it does not generate exercises at runtime.

## 2:38–2:48 — Closing

**Visual:** AIterval, Education category, public demo, and repository.

**Narration:** AIterval does not ask busy people to find more time. It turns time
they already lose into a learning habit.
