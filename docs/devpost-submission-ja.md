# AIterval — Devpost提出文案（日本語確認用）

## プロジェクト名・部門

**AIterval**  
**部門:** Education  
**タグライン:** AIの待ち時間を、英語リスニングの時間へ。

## 概要

AItervalは、AIの処理を待つ15〜90秒を1問の英語リスニング練習へ変える、
ローカルファーストのChrome拡張です。提出版は132問のオリジナル内蔵問題
だけを使い、OpenAI APIキーを必要とせず、学習進捗をブラウザ内に保ちます。

## 着想

国際サマースクールで、私は英語の講義や会話を十分に聞き取れませんでした。
研究者としてまとまった学習時間もありませんでした。一方で、AIツールの処理を
何度も待っていました。AItervalは、すでに失っている待ち時間を小さな学習習慣へ
変えます。

## 何をするか

AItervalは、停止ボタンやbusy状態など限定した意味的コントロールだけからAIの
処理状態を検知し、プロンプトや回答本文は読みません。一定時間後に1問だけを表示し、
AIが完了しても再生中の文を途中で止めず、完了通知を表示します。学習者は問題を最後まで
続けるか、すぐAIへ戻るかを選べます。

132問は実用・アカデミック英語のために事前作成されたオリジナル問題です。進捗、
復習予定、弱点、設定、回収した待ち時間はローカルに保存されます。実行時のAI問題
生成は提出製品に含まれません。

## 実装

WXT、React、TypeScript、Chrome Manifest V3、Next.js、Zod、Vitest、
Testing Library、Playwright、pnpm、Vercel、GitHub Actionsを使用しました。
ChatGPT、Claude、Geminiごとの独立アダプター、決定的な状態機械、Shadow DOM UI、
ブラウザ音声合成、バージョン付きローカル保存、実拡張の決定的テストを備えています。

## CodexとGPT-5.6との協働

人間は実体験、製品判断、1問だけの体験、Education部門のストーリー、プライバシー
規則を決めました。Codex with GPT-5.6は、それをアーキテクチャ、アダプター、状態
機械、UI、テスト、デプロイ、デバッグ、セキュリティレビュー、提出証拠へ落とし込み、
実Chromiumテストと改善を支援しました。

これは開発時の役割であり、実行時の製品機能ではありません。提出版は決定的な内蔵
問題だけを使い、OpenAI APIを呼び出しません。

The primary Codex task `/feedback` Session ID is submitted privately through
Devpost.

## 難しかったこと・成果

- 会話本文へ触れず、変化する三つのAI UIで処理状態を検知すること。
- AI完了通知と学習状態を分離し、再生中の文を途切れさせないこと。
- アカウントや外部APIなしで132問と復習機能を提供すること。
- 実Manifest V3拡張を決定的なサイトfixtureで検証すること。
- 公開デモ、公開ソース、検証可能なリリースZIPを完成させたこと。

## 学びと今後

強いAI製品には「使わない境界」も重要です。AItervalは待ち時間回収に必要な状態だけを
観察し、学習内容と進捗を決定的かつローカルに保ちます。今後は5名テスト、観察結果に
基づく改善、オリジナル問題とfixtureの追加、録音音声の検討、Chrome Web Store審査を
進めます。
