# AIterval — Devpost提出文案（日本語確認用）

## プロジェクト名・部門

**AIterval**  
**部門:** Education  
**タグライン:** AIの待ち時間を、次の国際講義に備える英語リスニング時間へ。

## 概要

AItervalは、ChatGPT、Codex、Claude、Geminiの処理中に15〜90秒のリスニング問題を1問だけ提示する、ローカルファーストのChrome拡張です。GPT-5.6が今後参加する講義の概要を検証済みの個別問題に変換するため、新しい勉強時間を確保せずに実際の講義へ備えられます。

## 着想

国際サマースクールで、私は英語の講義や会話を十分に聞き取れませんでした。一方、改善のためのまとまった学習時間もありませんでした。同じ時期、AIツールの処理を何度も待っていました。AItervalは、この二つの問題を結び付けます。

## 何をするか

会話内容ではなく、停止ボタンなど限定した意味的な状態だけからAIの処理中を検知し、短いリスニング問題を開始します。AIが完了すると音声を即座に止め、ユーザーを元の仕事へ戻します。132問のオリジナル内蔵問題、進捗、復習予定はローカルで動きます。

「講義からリスニング問題を生成」では、ユーザーが明示的に入力した講義タイトル・概要・希望条件だけをサーバーへ送ります。OpenAI Responses API上のGPT-5.6がStructured Outputsで既存スキーマに合う問題を作成し、サーバーで再検証します。エクスポートした問題パックを拡張側でも再検証し、その後のAI待ち時間で優先します。

## 実装

TypeScript、React、WXT、Chrome Manifest V3、Next.js App Router、Zod、OpenAI JavaScript SDK、Vitest、Testing Library、Playwrightを使用しました。GPT-5.6以外のモデルは拒否し、入力長、問題数、出力トークン、タイムアウト、再試行、時間当たり回数、一日上限を制限しています。APIキーとデモコードはサーバーだけに置きます。

## Codexとの協働

主要実装とBuild Weekアップグレードの大部分は、同じプライマリCodexタスクで行いました。人間は実体験、1問だけの体験、プライバシー境界、優先順位、Education部門のストーリーを決定しました。Codexは仕様をアーキテクチャ、アダプター、状態機械、スキーマ、UI、テスト、セキュリティ、デプロイ、提出証拠へ落とし込みました。

## GPT-5.6が必要な理由

固定問題だけでは、次の特定講義の用語や聞き取りパターンへ備えられません。GPT-5.6の検証済み出力が、そのまま拡張の中核である待ち時間の問題になります。公開サンプルは明確にサンプルと表示し、ライブ呼び出しとは主張しません。

## 難しかったこと・成果

- 会話本文へ触れず、変化する三つのAI UIで処理状態を検知すること。
- AI完了時に確実に音声を止め、作業を邪魔しないこと。
- ローカル／API不要の製品を保ったまま、サーバー限定の生成経路を追加すること。
- 拡張の権限やCSPを広げず、安全なJSONインポートを設計すること。
- 公開・ログイン不要のデモ、配布ZIP、実Chromium E2Eまで完成させたこと。

## 学びと今後

重要だったのは「AIを増やす」ことではなく、AIだけが価値を出せる境界を選ぶことでした。講義前の個別問題生成にはGPT-5.6を使い、検知・音声・中断・復習は高速で説明可能なローカル処理に保ちます。次は準備済みの5名テストを実施し、観察結果に基づいて改善し、Chrome Web Store審査へ進みます。
