# Human–Codex collaboration

## Origin and human direction

The creator supplied the real problem: at an international summer school, lectures and conversations in English were difficult to understand, yet dedicated study time was scarce. The creator noticed a second recurring problem—short waits while AI tools worked—and chose to connect them.

The human decisions include the one-question waiting-time loop, immediate pause
when AI work completes, local-first history, no punitive daily streak, Chrome as
the first platform, the Education category, 132 pre-authored exercises as the
submitted runtime, and the non-negotiable boundary that AIterval never reads AI
prompts or responses.

## What Codex contributed

Most core implementation and refinement were completed in this same primary
Codex task. Codex with GPT-5.6 translated the product brief into the workspace
architecture, semantic site adapters, deterministic state machine, schemas and
migrations, React surfaces, local recommendation logic, privacy controls, tests,
deployment configuration, and submission material. It also explored a runtime
generation path during development; the creator’s final decision archived that
path and kept the submitted product deterministic and pre-authored.

Codex also helped by running the real build and Chromium suites, diagnosing environment and locator failures, comparing extension permissions and CSP, checking official requirements, keeping claims evidence-based, and preparing reproducible release/deployment checks. The creator remained responsible for the lived insight, product priorities, final review, real user testing, private credentials, video narration/upload, and submission.

## Review boundary

Codex did not invent a user-test result, testimonial, runtime model claim,
deployment status, or model session identifier. Human-only steps remain
unchecked until performed.
